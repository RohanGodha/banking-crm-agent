# RM Copilot — Demo Video Script (~7 minutes)

A beat-by-beat guide: **when**, **what to show on screen**, and **what to say**.

---

## Pre-recording checklist

- [ ] Warm the backend: open `https://banking-crm-agent.onrender.com/healthz` once (free-tier cold start is ~50s, otherwise the first query feels slow).
- [ ] Confirm LLM quota is fresh — avoid running 10–15 test queries right before recording, or drafts may fall back to the mock. Record in a fresh window.
- [ ] To demonstrate WhatsApp send:
  - **Local (recommended):** numbers are already set in `frontend/.env.local` → run `npm run dev`, and stay logged into WhatsApp Web in the same browser.
  - **Live:** set `VITE_DEMO_PHONES` in Vercel and redeploy.
- [ ] Browser at 100% zoom, notifications off, record at 1080p.
- [ ] Start in dark theme (default); switch to light briefly mid-demo.
- [ ] Prepare two views: desktop, and a mobile view (DevTools device toolbar) for the responsive segment.

## Contingencies

- **Drafts show "mock":** the LLM quota is exhausted. Pause, wait 5–10 minutes, and confirm with a single call to `/tools/generate_whatsapp_message` (expect `llm_route=groq`) before resuming.
- **First query is slow:** the backend was cold. Run one throwaway query to warm it, then record.
- **WhatsApp shows "not on WhatsApp":** that customer is not mapped in `VITE_DEMO_PHONES`; use a mapped customer (Priya, Aarav, Ananya, Vikram, or Neha).
- **Backend unavailable:** record locally with `npm run dev` and `uvicorn` as a fallback.

## Delivery tips

- Speak slightly slowly; pause 1–2 seconds after each scenario to ease editing.
- Point with the cursor when mentioning "source", "Compliance OK", and "composite score".
- Target total length: **6–8 minutes** (the assignment allows 5–10; 7 is the sweet spot).
