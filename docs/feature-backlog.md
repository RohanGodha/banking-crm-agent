# Feature backlog — potential additions

Parked for later. Current build is feature-complete for the assignment; these are
prioritised enhancements drawn from the reference projects (VFS RAG bot, CHAI
devkit, insurance multi-agent, Text-Voice-Analysis) and BFSI best practice.

## Tier 1 — high impact, low effort
- [ ] **Confirm / Modify / Cancel flow** (from CHAI) — explicit human-in-the-loop gate before persisting/queuing an outreach batch, with a review summary card.
- [ ] **Global commands** — `help`, `back`, `cancel`, `reset` recognised at any turn.
- [ ] **Per-candidate "why now" one-liner** generated by the LLM from top features (richer than the current rule-based rationale).
- [ ] **Export** — download candidates + drafts as CSV/XLSX for the RM's records.
- [ ] **Tighten CORS** to the exact Vercel domain(s) in production.

## Tier 2 — strong product value
- [ ] **Full hybrid RAG on deploy** — enable Chroma (or Databricks Mosaic AI Vector Search) on a paid/again-free tier so dense+BM25 runs live, not BM25-only.
- [ ] **Trained ML propensity model** behind the existing `Scorer` interface; A/B vs the heuristic baseline; show model confidence.
- [ ] **Real WhatsApp send** via a `MessageChannel` port (Twilio/Meta Cloud API/Gupshup) with approved templates + opt-in/consent tracking.
- [ ] **Conversation summary + sentiment trend** persisted per session (from VFS bot's save_chat) for RM handover notes.
- [ ] **Adaptive-card-style structured replies** (from CHAI) for richer in-chat actions.

## Tier 3 — platform / scale
- [ ] **Multi-tenant + SSO** (JWT + refresh; per-RM row-level security in Databricks).
- [ ] **Databricks-native deployment path** — MLflow model registration + Mosaic AI Agents serving (mirrors CHAI devkit), as an alternative to Render.
- [ ] **Distributed tracing** (OTLP → Honeycomb/Grafana free) beyond the local `agent_traces` table.
- [ ] **Background re-embedding job** (Databricks Workflow/Airflow) for fresh interaction-note vectors.
- [ ] **Tool-result cache wired in** (schema exists) with TTL for repeat queries.

## Tier 4 — multimodal / analytics
- [ ] **Voice + call-transcript analysis** (from Text-Voice-Analysis) — ingest call recordings, transcribe, feed sentiment + intent into customer signals.
- [ ] **More languages** for drafts (full Indic set: Bengali, Punjabi, Kannada, Telugu, Odia, etc.) with per-language compliance dictionaries.
- [ ] **Inbound reply handling** — when a customer replies on WhatsApp, classify intent + suggest the RM's next step.
- [ ] **Campaign analytics dashboard** — approval rate, send rate, conversion tracking per campaign.

## Tier 5 — governance / BFSI hardening
- [ ] **Audit export** — signed, immutable trace bundle per campaign for compliance review.
- [ ] **PII redaction** layer on logs/traces.
- [ ] **Rate limiting + abuse protection** on the API.
- [ ] **Role-based prompts** — RM vs branch-manager vs product-owner personas.
- [ ] **Eval harness** (from CHAI's agent_evaluation) — golden-set grading of plan quality + draft quality with an LLM judge.
